# ADR-001: 技術スタック選定

- **日付**: 2026-03-10
- **ステータス**: 承認済み
- **チケット**: #4

## コンテキスト

SomedayPocketsの初期技術スタックを決定する必要がある。要件として：
- Webアプリケーション（PC利用が主）
- AI提案機能（マイルストーン提案）
- 個人開発・個人利用
- デプロイ先はCloudflare

## 決定

### フロントエンド
- **Next.js** (App Router) + **React**
- **shadcn/ui** + **Tailwind CSS** でUI構築
- Cloudflareへのデプロイは **@opennextjs/cloudflare** を使用

### バックエンド
- **Next.js API Routes**（Cloudflare Workers上で実行）

### データベース
- **Cloudflare D1** (SQLite互換)
- **Drizzle ORM** で型安全なDB操作

### 認証
- **Cloudflare Access**（ゼロトラスト認証。アプリ側の実装不要）

### AI
- **Claude API** でマイルストーン提案機能を実装

### インフラ
- **Cloudflare Pages** (フロント + Workers)
- **Cloudflare D1** (DB)
- **Cloudflare Access** (認証)

## 理由

- **Next.js**: React エコシステムの恩恵を最大限に受けられる。App Routerでサーバーコンポーネントを活用できる
- **OpenNext**: Next.jsをCloudflare上で動かすためのアダプタ。Vercel以外へのデプロイを可能にする
- **shadcn/ui**: コピペ型UIコンポーネント。依存が少なくカスタマイズしやすい
- **Cloudflare D1**: Workers と同じエコシステムで管理でき、無料枠が太い
- **Drizzle ORM**: D1との相性が良く、軽量で型安全
- **Cloudflare Access**: 個人利用なのでアプリ側に認証ロジックを持たせない。インフラ層で解決する
- **Claude API**: AI提案機能の要件に対して高品質な応答が期待できる
