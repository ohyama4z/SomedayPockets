# ADR-001: ✅ WebアプリのホスティングとしてCloudflare Pages + Next.js構成を採用する

- **日付**: 2026-03-10
- **ステータス**: 承認済み
- **チケット**: #4

## コンテキスト

SomedayPocketsの初期技術スタックを決定する。要件：
- PCブラウザで使うWebアプリ
- AI提案機能（マイルストーン提案）を持つ
- 個人開発・個人利用
- デプロイ先はCloudflare（無料枠・運用コスト重視）

## 検討した選択肢

### フレームワーク

#### 選択肢A: Next.js (App Router)
- **pros**: Reactエコシステムの恩恵を最大限に受けられる。サーバーコンポーネントでAPI・UIを一体管理できる。@opennextjs/cloudflareでCloudflare Workersへのデプロイが可能
- **cons**: Cloudflare環境との相性調整が必要（Node.js APIが一部使えない）

#### 選択肢B: Remix
- **pros**: Cloudflare Workers向けに最適化されており、Edge環境との相性が良い
- **cons**: Next.jsと比べてエコシステム・コミュニティが小さい。学習コストあり

#### 選択肢C: SvelteKit
- **pros**: 軽量・高速。Cloudflare Pagesへのデプロイが公式サポート
- **cons**: ReactベースのUIライブラリ（shadcn/ui等）が使えない。Reactと比べてエコシステムが小さい

### UIコンポーネント

#### 選択肢A: shadcn/ui + Tailwind CSS
- **pros**: コピペ型でバンドルサイズに影響しない。カスタマイズ自由度が高い。Tailwindとの親和性が高い
- **cons**: コンポーネントを自分のコードとして持つため、アップデートは手動

#### 選択肢B: MUI (Material UI)
- **pros**: コンポーネントが豊富。デザインシステムが確立されている
- **cons**: バンドルサイズが大きい。Tailwindとの共存が難しい

### データベース

#### 選択肢A: Cloudflare D1 + Drizzle ORM
- **pros**: Cloudflareエコシステム内で完結。無料枠が太い。SQLite互換で扱いやすい。DrizzleはD1との相性が良く型安全
- **cons**: SQLiteの制約を受ける（例: 複雑なJOINのパフォーマンス）。スケールアップの選択肢が限られる

#### 選択肢B: PlanetScale / Neon 等の外部DB
- **pros**: スケーラブル。MySQL/PostgreSQL互換で実績が多い
- **cons**: Cloudflare外のサービスへの依存が増える。無料枠の制約あり

### 認証

#### 選択肢A: Cloudflare Access
- **pros**: アプリ側に認証ロジックを持たせない。インフラ層で完結。個人利用に十分
- **cons**: Cloudflare依存。外部ユーザーへの公開には向かない

#### 選択肢B: NextAuth.js / Clerk 等
- **pros**: 柔軟なプロバイダー対応。アプリ内で認証フローを制御できる
- **cons**: 実装コストがかかる。個人利用アプリには過剰

## 決定

**Next.js (App Router) + shadcn/ui + Tailwind CSS + Cloudflare D1 + Drizzle ORM + Cloudflare Accessを採用する。**

### 理由

- **Next.js**: Reactエコシステムを活かしたい。shadcn/uiとの組み合わせが強力。@opennextjs/cloudflareでCloudflareデプロイも可能
- **shadcn/ui**: 個人開発でデザインに時間をかけたくない。カスタマイズしやすく、バンドルサイズを増やさない
- **Cloudflare D1 + Drizzle**: Cloudflare内で完結させてインフラを単純に保つ。個人利用規模ではSQLiteの制約は問題にならない
- **Cloudflare Access**: 個人利用のみなのでアプリ側に認証ロジックを持たせる必要がない。最もシンプルな選択
