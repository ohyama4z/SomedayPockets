# SomedayPockets 要件定義

> ステータス: ドラフト

## コア体験

タスクの「捕獲」と「組み立て」だけをユーザーが行い、それ以外（浮上判断・手順分解・種別分類）はシステムが担う。
開いたときに見えるのは「次にやること」と「システムからの提案」だけ。全体把握が必要なときはいつでも一覧に切り替えられる。

## 設計原則

1. **捕獲の摩擦をゼロにする** — タイトル一行で投げ込めること。メタデータ入力を強制しない
2. **ネクストはノイズゼロ** — 次にやることと浮上してきた提案だけが見える。沈殿物は表示しない
3. **棚卸しをユーザーに強いない** — 分解・浮上判断はシステムの責務
4. **やりたい/やるべきの区別を維持する** — 混在させるとリストがストレス源になる
5. **組み立ての自由を保証する** — 順序や構成はユーザーが決める。システムは素材を差し出す側に徹する
6. **全量アクセスを常に可能にする** — 普段は隠れていても、見たいときにはすべて一覧できること

## 三層構造

### Layer 1: インボックス（入口）

思いついた完了状態をすぐ投げ込む場所。

- タイトルのみで登録できる（最小入力）
- 即実行フラグ: 今すぐできるものはストックをスキップしてネクストに直行（任意）
- メタデータ（種別・期限など）の入力は捕獲時には求めない
- モバイルでも使える（PWA）

### Layer 2: ネクスト（出口）

組み立て済みの「次にやること」リスト。

- **表示要件**:
  - want / must / quick の区別が視覚的にわかる
  - want（やりたいこと）が主役として目立ち、must（やるべきこと）は補助的な見え方
  - quick は隙間に差し込めるものとして軽い存在感で表示
  - 今やれないものは一切表示しない
- **種別の変更**:
  - システムが自動分類した種別を、ネクスト内で簡単に変更できる（ワンタップ程度）
- **組み立てモード**:
  - ストックから浮上してきたもの + 自分で引き上げたものを配置する
  - 順序はユーザーが自由に決められる
  - いつでも組み立て可能（前日夜・当日朝など）
- **実行中**:
  - 完了したら消える
- **進捗管理**:
  - 全体のコンプリート率を表示（完了数/全数）

### Layer 3: ストック（中枢）

捕獲されたタスクが待機する場所。普段はユーザーの目に触れない。

- 捕獲されたタスクはデフォルトでここに入る（即実行フラグ付きを除く）
- ユーザーが能動的に棚卸しする必要はない
- **AI自動分解**:
  - ストックに入った時点で、システムが完了状態から手順への分解案を自動生成する
  - 分解と同時に種別（want / must / quick）も自動分類する
  - 分解案はストック内に保持されるが、自動では適用しない
  - ユーザーがネクストに引き上げる際に「こう分解できそうですが？」と提示し、採用・編集・無視を選べる
- **親子関係（2階層）**:
  - 完了状態（親）の下に手順（子タスク）をぶら下げられる
  - AI分解案を採用した場合、自動的に親子構造が作られる
  - 手動での子タスク追加・編集も可能
- **浮上条件**（タスクがネクスト候補として提示される条件）:
  - 分解完了: AI分解によりアクション可能な手順に落ちた時点で浮上
  - 期限接近: 期限付きタスクの期限が近づいたとき
  - 外部トリガー: 待ちタスクの条件成立時（ユーザーが手動で「トリガー到来」を記録。将来的に通知連携を検討）
- **再分解**:
  - 浮上後に着手されないタスクは、分解の仕方が適切でなかったと判断し再分解を試みる
- **全体ビュー**:
  - 見たいときにはすべてのタスクを一覧できる
  - タスク数の概況が把握できる

## タスクのライフサイクル

```
捕獲 → [即実行？] → Yes → ネクスト → 完了
                → No  → ストック
                          → AI自動分解（手順生成 + 種別分類）
                            → [分解完了 / 期限接近 / 外部トリガー] → 浮上
                              → 分解案を提示（採用/編集/無視）
                                → ネクスト → 完了
                                → [着手されない] → 再分解
```

---

## タスクの属性

| 属性 | 必須/任意 | 説明 |
|------|-----------|------|
| タイトル | 必須 | 完了状態を表す短文（例:「福岡の宿をとる」） |
| 種別 | 自動 | want / must / quick。AI分類後にユーザーが変更可能 |
| 期限 | 任意 | 設定時は浮上条件に使用 |
| トリガー条件 | 任意 | 待ちタスクの実行可能条件メモ |
| ステータス | 自動 | ストック中 / 浮上済み / ネクスト / 完了 |
| 子タスク | 自動/手動 | AI分解案またはユーザー手動による手順リスト |

## プラットフォーム

- Webアプリケーション（PWA）として開発
- モバイルでも利用可能

## 未決事項

- AI分解の精度が低い場合のフォールバック体験（実運用で検証しながら決定）
