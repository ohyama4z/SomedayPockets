# SomedayPockets 要件定義

> ステータス: ドラフト

## コア体験

タスクの「捕獲」と「組み立て」だけをユーザーが行い、それ以外（鮮度管理・浮上判断・手順分解）はシステムが担う。
開いたときに見えるのは「次にやること」と「システムからの提案」だけ。全体把握が必要なときはいつでも一覧に切り替えられる。

設計の背景は [設計思想](設計思想.md) を参照。

---

## 三層構造

### Layer 1: インボックス（入口）

思いついた完了状態をすぐ投げ込む場所。

- タイトルのみで登録できる（最小入力）
- 以下のメタデータは任意で付与可能:
  - 種別: want（やりたい）/ must（やるべき）/ quick（隙間で片付く単発）
  - 即実行フラグ: 今すぐできるものはストックをスキップしてネクストに直行
- モバイルからでもすぐ書ける（モバイルファースト）

### Layer 2: ネクスト（出口）

組み立て済みの「次にやること」リスト。

- **表示要件**:
  - want / must / quick の区別が視覚的にわかる
  - want（やりたいこと）が主役として目立ち、must（やるべきこと）は補助的な見え方
  - quick は隙間に差し込めるものとして軽い存在感で表示
  - 今やれないものは一切表示しない
- **組み立てモード**:
  - ストックから浮上してきたもの + 自分で引き上げたものを配置する
  - 順序はユーザーが自由に決められる
  - いつでも組み立て可能（前日夜・当日朝など）
- **実行中**:
  - 完了したら消える
- **進捗管理**:
  - 全体のコンプリート率を表示（完了数/全数）

### Layer 3: ストック（中枢）

捕獲されたタスクが待機する場所。普段はユーザーの目に触れない。

- 捕獲されたタスクはデフォルトでここに入る（即実行フラグ付きを除く）
- ユーザーが能動的に棚卸しする必要はない
- **AI自動分解**:
  - ストックに入った時点で、システムが完了状態から手順への分解案を自動生成する
  - 分解案はストック内に保持されるが、自動では適用しない
  - ユーザーがネクストに引き上げる際に「こう分解できそうですが？」と提示し、採用・編集・無視を選べる
- **親子関係（2階層）**:
  - 完了状態（親）の下に手順（子タスク）をぶら下げられる
  - AI分解案を採用した場合、自動的に親子構造が作られる
  - 手動での子タスク追加・編集も可能
- **浮上条件**（タスクがネクスト候補として提示される条件）:
  - 期限接近: 期限付きタスクの期限が近づいたとき
  - 鮮度アラート: 一定期間放置されたタスクが劣化する前に通知
  - 外部トリガー: 待ちタスクの条件成立時（ユーザーが手動で「トリガー到来」を記録）
  - 定期浮上: 期限のない義務を低頻度で差し出す（「今週これはどう？」）
- **全体ビュー**:
  - 見たいときにはすべてのタスクを一覧できる
  - タスク数や鮮度状態の概況が把握できる

---

## タスクのライフサイクル

```
捕獲 → [即実行？] → Yes → ネクスト → 完了
                → No  → ストック（AI分解案を自動生成）
                          → [浮上条件 or ユーザー引き上げ]
                            → 分解案を提示（採用/編集/無視）
                              → ネクスト → 完了
```

---

## タスクの属性

| 属性 | 必須/任意 | 説明 |
|------|-----------|------|
| タイトル | 必須 | 完了状態を表す短文（例:「福岡の宿をとる」） |
| 種別 | 任意 | want / must / quick。未指定でも可 |
| 期限 | 任意 | 設定時は浮上条件に使用 |
| 鮮度 | 自動 | 捕獲日時から自動計算。経過に応じて劣化表示 |
| トリガー条件 | 任意 | 待ちタスクの実行可能条件メモ |
| ステータス | 自動 | ストック中 / 浮上済み / ネクスト / 完了 |
| 子タスク | 任意 | AI分解案またはユーザー手動による手順リスト |

---

## 未決事項

- 浮上条件の具体的な閾値（鮮度は何日で警告とするか）
- want/must/quick の判定タイミング（捕獲時に聞くか、後から変更可能にするか）
- 外部トリガーの入力方法（手動のみか、通知連携か）
- プラットフォーム選定（モバイルアプリ / PWA / 既存ツール拡張）
- 親子関係の表示形式（視認性重視、ツリー形式は避ける）
- AI分解の精度が低い場合のフォールバック体験
