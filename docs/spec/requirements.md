# SomedayPockets 要件定義

> ステータス: 確定（v1）

## コア体験

暇な日にアプリを開いたら「これやろう」とテンションが上がる体験を提供する。
「やりたいこと」も「やらなきゃいけないこと」も一元管理し、時間が空いたらこのアプリを見ればよい状態をつくる。

## 1. コア機能と基本設計

* **アイテム管理:** リストの追加、完了、完了済み表示。
* **種別:** 「やりたい」と「やらなきゃ」の2種別で区別する。ジャンル分け（カテゴリ・タグ）はMVP時点では導入しない。
* **親子関係（2階層）:** 長期目標の下に具体タスクをぶら下げられる。表示はツリー形式を避け、視認性を重視する。
* **期限設定:** アイテムごとの期限設定機能。
* **進捗管理:**
  * 長期目標単位: 子タスクの進捗率を表示
  * 全体: コンプリート率を表示（完了数/全数。母数が増え続ける中で積み上がっていく形）

## 2. 入力と優先順位付け

* **入力負荷軽減:** アイテム作成時、ユーザーの負担を考慮しタイトル入力のみで完了。追加の気軽さを最優先する。
* **優先順位の提案:** 作成後、システム側から任意のタイミング（例：ポップアップ）で優先順位の入力を促す仕組みを導入。AIによる自動設定は行わず、ユーザーの意思を重視。

## 3. AI提案機能

* **マイルストーン提案:** 長期目標に対するマイルストーンや具体的なアクションプランをAIが提案する。提案結果は子タスクとして追加できる。

## 4. プラットフォーム

* **メインプラットフォーム:** パソコンでの利用を主とし、Webアプリケーションとして開発。
* **Appleリマインダー連携:** 初期スコープでは対象外（将来検討）。
