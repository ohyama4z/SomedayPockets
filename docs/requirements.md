# MyQuest 要件定義

> ステータス: ドラフト（議論中）

## 1. コア機能と基本設計

* **To Do管理:** リストの追加、完了、完了済み表示。
* **期限設定:** タスクごとの期限設定機能。
* **分類:** カテゴリ分け、タグ付け、ジャンル別の表示。一括確認機能も含む。
* **進捗管理:** モチベーション維持のため、プログレスバーやコンプリート率を視覚的に表示。

## 2. 入力と優先順位付け

* **入力負荷軽減:** タスク作成時、ユーザーの負担を考慮しタイトル入力のみで完了。
* **優先順位の提案:** 作成後、システム側から任意のタイミング（例：ポップアップ）で優先順位の入力を促す仕組みを導入。AIによる自動設定は行わず、ユーザーの意思を重視。

## 3. AI提案機能

* **新しい提案:** 長期目標に対するマイルストーンや具体的なアクションプランをAIが提案する。

## 4. 同期と連携

* **メインプラットフォーム:** パソコンでの利用を主とし、Webアプリケーションとして開発。
* **Appleリマインダー連携:** Appleクラウド経由での双方向同期を実現。Web側で追加・完了したタスクはAppleリマインダーに、Appleリマインダーでの変更はWebアプリに反映されるように設計。
