# プロセスレビュー

サブエージェント（第三者）を起動し、プロセス・運用の改善点を指摘させる。
改善点があれば `/propose` で起票する。なければ一言報告して終わる。

## 1. 参照先パスを収集

```bash
ls knowledge/
ls notes/
find ~/.claude/projects -name "MEMORY.md" 2>/dev/null
```

## 2. サブエージェントを起動

Agentツールで以下のプロンプトを渡す（収集したパス情報を含める）：

```
あなたはこのプロジェクトの開発プロセスを外から観察しているシニアエンジニアです。
以下のファイルを自分で読み込んで、プロセス・運用の改善点を指摘してください。

【読むべきファイル】
- CLAUDE.md（運用ルール）
- .claude/skills/*/SKILL.md（スキル一覧）
- .claude/settings.json（設定）
- docs/decisions/（ADR一覧）
- knowledge/（知見。ファイル一覧: [ls knowledge/ の結果]）
- notes/（作業ログ。ファイル一覧: [ls notes/ の結果]）
- Auto Memory（[find結果のパス]）
- git log --oneline -20（直近の作業履歴）
- gh issue list --repo ohyama4z/SomedayPockets --state all --limit 10

【評価観点】
- スキルや運用ルールで繰り返し手作業が発生していないか
- 曖昧・未定義なルールがあって判断に迷いそうな箇所はないか
- 情報が揮発していてセッション再開時にコンテキストが失われそうな箇所はないか
- スキルの使い勝手・呼び出しタイミングに改善余地はないか
- ドキュメントに実態と乖離が生じていないか

改善点がある場合は、タイトルと背景・内容・期待する効果を具体的に述べてください。
なければ「特に改善点なし」と答えてください。
```

## 3. 提案または報告

**改善点がある場合**: サブエージェントの指摘をもとに `/propose` スキルを呼んでIssueを起票する。複数あれば複数起票してよい。

**特になければ**: 「✅ プロセスレビュー完了：特に改善点なし」と一言報告する。
