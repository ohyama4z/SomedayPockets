# PreToolUseフックでコマンドガードレール

- **日付**: 2026-04-17
- **チケット**: #96
- **作業**: prefix一致でallow設定できないコマンドパターンの排除

## 知見

### permissions.allowのprefix一致の限界
`permissions.allow`はprefix一致のため、以下のパターンはallow設定が困難：
- `cat > file` — `cat`が広すぎる
- `git -C <動的パス>` — パスが動的
- `gh api` — GraphQL等で広すぎる

これらはスキルの指示文で禁止しても強制力がない（CLAUDE.mdルールと同様）。

### PreToolUseフックでガードレールを敷く
`settings.json`の`hooks.PreToolUse`でBashツールの実行前にスクリプトを呼び出し、禁止パターンを検出・ブロックできる。

- stdinからJSON（`tool_input.command`）を受け取る
- exit 0 + `permissionDecision: "deny"` のJSONを返すとブロック
- exit 2 + stderrメッセージでもブロック可能（簡易版）
- `jq`でJSONパース・生成する

### 代替パターン
| 禁止パターン | 代替 |
|-------------|------|
| `cat > tmp/file` → `--body-file` | `--body "$(cat <<'EOF'...EOF)"` ヒアドキュメント直接渡し |
| `git commit -F tmp/file` | `git commit -m "$(cat <<'EOF'...EOF)"` |
| `git -C <path>` | スクリプト内に閉じ込める（`bash .claude/scripts/*.sh`でallow） |
| `gh api graphql` | スクリプトにラップ（`add-sub-issue.sh`等） |
