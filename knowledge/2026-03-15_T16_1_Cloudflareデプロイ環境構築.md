# Cloudflare デプロイ環境構築

- **日付**: 2026-03-15
- **チケット**: #16
- **作業**: @opennextjs/cloudflare導入とCloudflare Pagesへの初回デプロイ

## 知見

- `@opennextjs/cloudflare` は Next.js 16 に対応済み。導入は `npm install @opennextjs/cloudflare wrangler` のみでOK
- 設定ファイルは `wrangler.jsonc`（wrangler.tomlではなくJSON形式も可）、`open-next.config.ts`、`.dev.vars` の3つが必要
- `next.config.ts` に `initOpenNextCloudflareForDev()` を追加することで開発時のバインディング統合が有効になる
- ビルドは `opennextjs-cloudflare build`、デプロイは `opennextjs-cloudflare deploy` のCLIコマンドで実行
- SSH環境では `wrangler login` が使えない（ブラウザが開けない）。`CLOUDFLARE_API_TOKEN` 環境変数にAPIトークンを設定して認証する
- APIトークン設定時に `wrangler login` を実行すると `You are logged in with an API Token. Unset the CLOUDFLARE_API_TOKEN...` エラーになる。トークン認証とOAuth認証は排他的
- デプロイ先は `https://<name>.ohyama4z000.workers.dev`（wrangler.jsoncの `name` フィールドがサブドメインになる）
