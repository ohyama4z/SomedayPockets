## 知見

### スキルファイル（.claude/skills/*/SKILL.md）のフロントマター

サポートされる属性は限られている。`allowed-tools` は**サポート外**（警告が出るだけで動作はするが意味がない）。

サポートされる属性:
- `name` - スキル名
- `description` - 説明（Skill toolでの説明文に使われる）
- `argument-hint` - 引数のヒント
- `disable-model-invocation` - trueにするとSkill toolから呼べなくなる（**削除推奨**）
- `user-invocable`, `compatibility`, `license`, `metadata`

### カスタムサブエージェント（.claude/agents/*.md）

`.claude/agents/` にマークダウンファイルを置くことでカスタムサブエージェントを定義できる。

```markdown
---
name: エージェント名
description: 説明
---

システムプロンプト（エージェントの役割・手順・出力フォーマットを定義）
```

Agentツールで `subagent_type: エージェント名` を指定して起動する。会話履歴を持たない独立したコンテキストで動くため、第三者視点のレビューなどに適している。

### Stopフックによる定期プロンプト

`settings.json` の `hooks.Stop` に `type: "prompt"` のフックを設定すると、Claudeの応答終了ごとにシェルコマンドを実行し、その出力をClaudeへのプロンプトとして注入できる。

```json
"hooks": {
  "Stop": [{
    "hooks": [{ "type": "prompt", "command": "bash script.sh" }]
  }]
}
```

スクリプトが何も出力しなければ何も起きない。N回に1回だけ出力するカウンタースクリプトと組み合わせることで、定期的なレビュー促進などが実現できる。

### settings.json の permissions.allow

スキル実行中に使うBashコマンドも `permissions.allow` に含めておく必要がある。`git push` や `gh pr *` など漏れやすいので注意。
