# Claude Code権限設計とAI自律実行の制御

- **日付**: 2026-03-14
- **チケット**: #36
- **作業**: settings.jsonのpermissions整備

## 知見

### CLAUDE.mdのルールには強制力がない
「AI自律実行の判断基準」をCLAUDE.mdに文章で書いても、AIがそのルールを守るかどうかは保証されない。
強制力を持たせるには `settings.json` の `permissions.allow` / `deny` で制御する必要がある。

### permissions の設計原則
- `allow` に入っている操作 → 確認なしで自動実行
- `deny` に入っている操作 → ブロック（allowより優先）
- どちらにも入っていない操作 → ユーザーに確認プロンプトが出る

### `&&` チェーンはallowをすり抜ける
`Bash(git add *)` が allowに入っていても、`git add foo && git commit -m "msg"` はマッチしない。
Claude Codeはシェル演算子（`&&`）を認識しており、プレフィックスマッチが `&&` をまたぐことを明示的に禁止している。
→ コマンドはチェーンせず、Bash呼び出しを分けることで対応する。

### gh コマンドは削除・クローズ系をdenyで守る
`Bash(gh issue delete *)` や `Bash(gh pr close *)` は明示的にdenyに入れておく。
`gh issue *` を一括allowすると意図せず削除が通ってしまうリスクがある。

### Agent Teamsの有効化
`settings.json` の `env` ブロックに `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS: "1"` を追加し、Claude Codeを再起動することで有効になる。
`teammateMode: "in-process"` も合わせて設定する。
