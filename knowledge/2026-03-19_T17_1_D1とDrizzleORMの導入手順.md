# D1 と Drizzle ORM の導入手順

## D1データベース作成
- `CLOUDFLARE_ACCOUNT_ID=<id> npx wrangler d1 create <db名>` で作成
- APIトークンに **Account > D1 > Edit** 権限が必要（Cloudflareダッシュボードで設定）
- `database_id` はシークレットではないので `wrangler.jsonc` にコミットしてよい

## wrangler.jsonc の設定
- `d1_databases` にバインディングを追加
- `migrations_dir` を Drizzle のデフォルト出力先 `"drizzle"` に指定する（wrangler のデフォルトは `migrations/`）

## Drizzle ORM
- `drizzle-orm`（本体）と `drizzle-kit`（マイグレーションツール、devDependency）をインストール
- `drizzle.config.ts` で `dialect: "sqlite"` を指定（D1 は SQLite ベース）
- スキーマは `drizzle-orm/sqlite-core` の `sqliteTable` で定義
- 自己参照FK: `.references(() => テーブル.カラム)` で定義可能

## マイグレーション
- `npx drizzle-kit generate` で SQL 生成
- `npx wrangler d1 migrations apply <db名> --remote` でリモート適用
- `npx wrangler d1 migrations apply <db名>` でローカル適用

## Next.js (App Router) からの接続
- `@opennextjs/cloudflare` の `getCloudflareContext()` で `env.DB` を取得
- `drizzle(env.DB, { schema })` で Drizzle インスタンスを生成
- ローカル開発では `next.config.ts` で `initOpenNextCloudflareForDev()` が必要（既に設定済み）
- `env.d.ts` で `CloudflareEnv` インターフェースに `DB: D1Database` を宣言

## .gitignore
- `.wrangler/` を追加（ローカルD1のSQLiteファイル等が含まれる）
